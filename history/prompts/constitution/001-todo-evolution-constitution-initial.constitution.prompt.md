## Prompt

# Todo Evolution Project – Spec Constitution

## 1. Purpose

This Constitution defines the governing principles, constraints, and development rules for the **5-Phase Evolution of Todo Project**.
The project evolves from a simple in-memory Todo application to an AI-powered, cloud-native system deployed on Kubernetes.

All development must strictly follow **Spec-Driven Development (SDD)** using **Spec-Kit Plus** and **Claude Code**.

---

## 2. Core Principles

### 2.1 Spec-Driven Development Only
- Every feature **must** be defined in a written Markdown **Specification** before implementation.
- No implementation may begin without an approved spec.
- All code must be generated by **Claude Code**.
- Manual code writing, editing, or patching is **strictly forbidden**.
- If generated code is incorrect, the **spec must be refined**, not the code.

### 2.2 Incremental Evolution
- The project evolves sequentially across **five phases**.
- Each phase builds on the previous one without breaking existing functionality.
- Backward compatibility must be preserved unless explicitly stated in the spec.

### 2.3 Single Source of Truth
- The Markdown specifications and this Constitution are the **authoritative source** for:
  - System behavior
  - Business logic
  - API contracts
  - Infrastructure design
- If code behavior differs from the spec, the spec prevails.

---

## 3. Phase Structure & Scope

### Phase I – In-Memory Console Application
**Technology Stack**: Python, Claude Code, Spec-Kit Plus
**Scope**: In-memory task storage, Console-based interaction, Core Todo functionality only

### Phase II – Full-Stack Web Application
**Technology Stack**: Next.js (Frontend), FastAPI (Backend), SQLModel, Neon Database (PostgreSQL)
**Scope**: Persistent task storage, REST API, Web UI for task management

### Phase III – AI-Powered Todo Chatbot
**Technology Stack**: OpenAI ChatKit, OpenAI Agents SDK, Official MCP SDK
**Scope**: Conversational interface, Natural language commands, AI agent orchestrating via APIs

### Phase IV – Local Kubernetes Deployment
**Technology Stack**: Docker, Minikube, Helm, kubectl-ai, kagent
**Scope**: Containerized services, Local Kubernetes deployment, Helm charts

### Phase V – Advanced Cloud Deployment
**Technology Stack**: DigitalOcean Kubernetes (DOKS), Kafka, Dapr
**Scope**: Cloud-native deployment, Event-driven architecture, Production-grade observability

---

## 4. Todo Feature Evolution

- Basic Features (Phase I+): Add, Delete, Update, View, Mark Complete
- Intermediate Features (Phase II+): Priorities, Tags, Search, Filter, Sort
- Advanced Features (Phase III+): Recurring tasks, Due dates, Reminders, AI rescheduling
